## 🎯 Project Overview

This project builds a complete customer churn prediction system for a subscription business:

- Generates a realistic 50K-record customer dataset
- Engineers RFM-inspired features (recency, frequency, monetary signals)
- Trains a **Random Forest** classifier with **SMOTE** class balancing
- Compares against a logistic regression baseline (+15pp accuracy gain)
- Visualizes churn risk, revenue trends, and KPIs in a **Streamlit dashboard**

## 🚀 Quick Start

### 1. Clone and install
```bash
git clone https://github.com/SwaroopBatchu/churn-prediction.git
cd churn-prediction
pip install -r requirements.txt
```

### 2. Generate sample data
```bash
python src/generate_data.py
```

### 3. Train the model
```bash
cd src
python train.py
```

### 4. Launch the dashboard
```bash
streamlit run src/dashboard.py
```

## 📊 Model Performance

| Model | Accuracy | ROC-AUC |
|---|---|---|
| Logistic Regression (baseline) | ~72% | ~0.74 |
| **Random Forest + SMOTE** | **~87%** | **~0.91** |

**Minority class (churn) recall improved by 18%** after SMOTE class balancing.

## 🔧 Feature Engineering

| Feature | Description |
|---|---|
| `tenure_segment` | Customer lifetime bucket (New / Growing / Established / Loyal) |
| `charge_ratio` | Monthly charge as % of average historical spend |
| `high_risk_flag` | Binary flag: high support calls + high payment delay |
| `revenue_tier` | Quartile-based revenue segment (Budget → VIP) |

---

## 📈 Dashboard Features

- **4 KPI cards** — total customers, churn rate, high-risk count, avg revenue
- **Risk tier distribution** — Low / Medium / High breakdown
- **Contract type churn analysis** — month-to-month vs annual
- **Revenue at risk** — monthly revenue by risk tier
- **Tenure segment churn rates** — loyalty vs new customer patterns
- **High-risk customer table** — top 20 accounts needing intervention
- **CSV export** — download scored customer list
